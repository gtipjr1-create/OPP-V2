# OPP - MASTER CONTEXT
## Product Identity
OPP is a personal operating system built to reduce noise and help me live, build, train, and think with clarity, discipline, and control.

It is the structure used to organize life, work, training, study, writing, and long-term building into one clear, disciplined framework.

It is not:
- a motivational system
- a productivity-hack playground
- a trendy self-improvement dashboard
- a cluttered life admin tool

It is:
- a structured personal operating framework
- a clarity system for disciplined living
- a control layer for long-term self-direction
- a place where priorities, standards, and execution stay aligned

---

## Core Purpose
OPP exists to reduce internal and external noise.

Its purpose is to help create:
- clarity over confusion
- structure over drift
- deliberate action over reactive living
- consistency over scattered effort
- long-term alignment over short-term impulse

---

## Core Philosophy
Life works better when it is structured intentionally.

OPP is built on the belief that:
- discipline creates freedom
- clarity reduces friction
- structure protects attention
- fewer better systems outperform constant reinvention
- self-respect is reinforced through consistent execution
- progress is built through repeatable standards, not emotional swings

## Core User
Primary user:
- self-directed builder

- the owner/operator using OPP for personal life architecture

---

## Core Domains
OPP tracks six domains of a disciplined life. These are locked for v1.

| Domain | Covers |
|---|---|
| Health | Training, body, recovery, physical standards |
| Work | Income, delivery, professional execution |
| Build | Products, projects, long-term creation |
| Mind | Learning, study, reading, thinking |
| Write | Writing, publishing, expression |
| Life | Relationships, environment, personal admin |

Planning and review are system behaviors, not domains.
Standards and routines are a reference layer, not a domain.
Long-term direction sits above the domains as the identity/direction layer.

These domains should feel connected, not fragmented.

---

## Product Positioning
OPP is the control layer above daily action.

Where other systems help with isolated tasks, OPP helps maintain:
- overall direction
- domain clarity
- execution standards
- decision consistency
- life architecture

It is a personal command center, not just a generic productivity app.

---

## Layer Contract (v1 Locked)
- Today: re-entry, daily order, and day-item execution surface.
- Domains: structural life lanes with Active/Steady state.
- Priorities: focus-management layer for selected commitments.
- Standards: governing rules that hold across mood, energy, and circumstances.
- Settings: account access and system language glossary.

Key distinction:
- Day items shape the current day.
- Commitments shape active focus.

## Architecture Principle
OPP should be modular but unified.

It must avoid:
- fragmented systems
- duplicate views of the same truth
- too many disconnected dashboards
- feature sprawl
- “nice idea” accumulation without execution value

Each part of OPP should reinforce one larger system of living.
