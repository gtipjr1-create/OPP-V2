# OPP Freeze Boundary

## Protected Areas
- Core screen-role model (Today, Domains, Priorities/Focus, Standards, Weekly Anchors)
- Vocabulary and meaning boundaries
- Existing mobile shell behavior and interaction baseline
- Supabase-backed data/auth flow unless explicitly planned

## No Rebuild Rule
Do not rebuild OPP from scratch or perform sweeping architecture rewrites during parked state maintenance. Prefer small, explicit, reversible changes only when authorized.

## No Forge Merge Rule
Do not merge OPP concept or structure into Forge. Keep OPP as a distinct product identity and codebase direction.

## No Productivity-Clone Drift
Do not drift into generic task-manager, habit-app, or dashboard-clone patterns that weaken OPP's command-center model.

## No Premature Supabase Expansion
Do not expand Supabase scope (new schema breadth, complex data layers, or infra-heavy additions) unless explicitly planned and requested.

## Current Status Rule
OPP is currently parked, not abandoned. Default work mode is preservation, organization, and verification unless explicit direction is given to expand product scope.
