# OPP Design System

## Visual Principles
- Calm, structured, and intentional
- Signal over decoration
- Strong hierarchy, low noise
- Premium but restrained presentation

## Spacing, Card, and Hierarchy Rules
- Consistent vertical rhythm across all core screens
- Distinct card surfaces for major content groups
- Headings and metadata must preserve scan order and role clarity
- Maintain clear separation between orientation content and management controls

## Tone of the App
- Disciplined
- Clear
- Grounded
- Operational
- Never loud, gamified, or novelty-driven

## Mobile / Desktop Assumptions
- Mobile-first behavior is a core baseline
- Fixed app chrome (header/nav) with bounded middle scroll surface
- Safe-area awareness is required for phone ergonomics
- Desktop should preserve the same conceptual hierarchy, not introduce a different product model

## Consistency Rules
- Preserve naming and semantic consistency across screens
- Keep Active/Steady, Day Item, Commitment, and Standard meanings stable
- Avoid duplicated UI blocks unless they provide clear orientation value
- Do not introduce style changes that blur screen roles or increase cognitive noise
