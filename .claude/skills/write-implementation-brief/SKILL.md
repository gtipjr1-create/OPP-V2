# Purpose

Use this skill to convert rough OPP product thinking into a clean, build-ready implementation brief.

OPP is a Personal Command Center for disciplined living and focused building.
It is not a generic productivity app or loose idea space.

This skill should help translate abstract thinking into structured product direction that is clear enough to implement deliberately.

# Use this skill when

Use this skill when:
- turning a rough idea into a product plan
- preparing for a screen rebuild or refinement pass
- clarifying the purpose of a new feature
- reducing ambiguity before implementation
- defining scope for a focused build session
- shaping product thoughts into something actionable

# Core standards

Apply these standards during brief creation:

1. Start from product purpose, not feature novelty.
2. Define the role of the screen or feature clearly.
3. Keep scope focused and appropriate for the current pass.
4. Protect OPP’s command-center identity.
5. Prefer structural clarity over unnecessary expansion.
6. Make the brief useful for implementation, not just discussion.
7. Highlight constraints and risks early.
8. End with a direct next move.

# Workflow

When using this skill:

1. Identify the core product problem or opportunity.
2. Define what the screen, feature, or refinement is meant to do.
3. Clarify why it matters inside OPP specifically.
4. Define the desired product outcome.
5. Define the scope for this pass.
6. Identify constraints, dependencies, and risks.
7. State what should not be changed or diluted.
8. End with a focused implementation direction.

# Output format

Structure the response like this:

## Problem
State the product problem clearly.

## Product role
Explain the intended role of the screen, feature, or refinement.

## Desired outcome
Describe what a good result should look like.

## Scope for this pass
Define what should be included now.

## Constraints
List boundaries, product truths, or implementation limits.

## Risks
Call out what could go wrong or create drift.

## Recommended implementation direction
State the clearest next move.

# Constraints

Do not:
- write vague brainstorm language
- suggest broad feature sprawl without reason
- ignore OPP’s core structure and terminology
- over-scope the current pass
- confuse product thinking with final implementation details unless requested
