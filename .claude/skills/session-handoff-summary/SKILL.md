# Purpose

Use this skill to create a clean end-of-session handoff for OPP work.

OPP is a Personal Command Center for disciplined living and focused building.
This skill should help preserve continuity between sessions by capturing what was accomplished, what became clearer, what still feels unresolved, and what the next refinement or implementation move should be.

# Use this skill when

Use this skill when:
- ending a product or build session
- summarizing screen refinements
- wrapping up a backend or structure pass
- capturing what changed before stopping work
- preparing for the next session
- reducing restart friction later

# Core standards

Apply these standards during summary writing:

1. Be clear and direct.
2. Capture real progress, not vague effort.
3. Note what became clearer in the product.
4. Identify what still feels weak, unresolved, or at risk.
5. Preserve OPP’s product identity and structure in the summary.
6. Keep the handoff easy to read and useful for restarting work.
7. End with the clearest next move.
8. Prefer signal over volume.

# Workflow

When using this skill:

1. Summarize what was completed or improved.
2. State what became clearer about the product or screen.
3. Identify what still needs refinement, implementation, or review.
4. Note any risks, ambiguities, or areas to protect.
5. Recommend the most important next step.

# Output format

Structure the response like this:

## What changed
Summarize the key work completed.

## What got clearer
State what improved in product understanding, structure, or direction.

## What still needs work
Identify unresolved issues or weak areas.

## Risks or watchouts
Call out anything that could cause drift or confusion later.

## Recommended next move
State the clearest next step directly.

# Constraints

Do not:
- write overly long summaries
- use vague “good progress” language without specifics
- turn the handoff into a brainstorm
- ignore unresolved problems
- recommend too many next steps at once
