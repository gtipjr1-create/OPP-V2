# Purpose

Use this skill to refine OPP screens that already have a working foundation but still need sharper structure, cleaner hierarchy, and better product clarity.

OPP is a Personal Command Center for disciplined living and focused building.
It should feel deliberate, structured, and meaningful — not generic, cluttered, or visually noisy.

This skill is for tightening and elevating a screen without losing the product’s identity.

# Use this skill when

Use this skill when:
- a screen works but feels rough
- hierarchy feels weak or unclear
- spacing feels off
- labels need sharpening
- the screen feels too generic
- a screen needs cleanup after backend or feature wiring
- refinement is needed before adding more features

# Core standards

Apply these standards during refinement:

1. Preserve the screen’s core role.
2. Strengthen hierarchy before adding anything new.
3. Reduce clutter and dead weight.
4. Improve spacing and rhythm.
5. Sharpen labels and meaning.
6. Protect OPP’s command-center identity.
7. Prefer clarity, structure, and deliberate use over decorative treatment.
8. Refine what exists before expanding scope.

# Workflow

When using this skill:

1. Identify the screen’s intended role.
2. Review its hierarchy and what draws attention first.
3. Check for clutter, weak spacing, or structural drift.
4. Review labels and terminology for clarity.
5. Identify anything that feels generic or off-positioning.
6. Separate findings into:
   - what should stay
   - what should be refined
   - what should be reduced or removed
7. End with the clearest next refinement move.

# Output format

Structure the response like this:

## Screen role
State what the screen should be doing.

## What is working
Identify what should be preserved.

## What feels weak
Call out hierarchy, spacing, terminology, or structure issues.

## What to refine
Give practical refinement directions.

## What to reduce or remove
Identify clutter, generic patterns, or unnecessary elements.

## Recommended next pass
State the clearest next refinement target.

# Constraints

Do not:
- drift into random redesign without reason
- add features just to make the screen feel fuller
- focus only on visuals while ignoring product meaning
- weaken OPP’s command-center identity
- suggest changes that create overlap with other layers
