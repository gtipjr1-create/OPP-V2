# Purpose

Use this skill to critique OPP from an information architecture and product-clarity perspective.

OPP is a Personal Command Center for disciplined living and focused building.
It is not a generic productivity app, casual task manager, or habit tracker clone.

This skill should help determine whether a screen, flow, or product decision strengthens that command-center identity or weakens it.

# Use this skill when

Use this skill when:
- reviewing OPP screens
- assessing navigation clarity
- checking whether a screen has a clear role
- identifying overlap between Today, Priorities, Domains, Standards, and This Week
- evaluating labels and terminology
- deciding whether a feature feels too generic or off-positioning
- refining product structure before implementation

# Core standards

Apply these standards during critique:

1. Every screen should have a clear and distinct role.
2. OPP should feel like a command center, not a generic productivity tool.
3. Labels should be precise, meaningful, and system-consistent.
4. Navigation should reinforce the product model, not blur it.
5. Features should support clarity, control, standards, priorities, and deliberate execution.
6. Avoid generic productivity patterns when they weaken product identity.
7. Prefer structural clarity over feature expansion.
8. Recommend refinement before adding unnecessary complexity.

# Review workflow

When using this skill:

1. Identify the purpose of the screen, flow, or feature.
2. Check whether that purpose is immediately clear.
3. Look for overlap with other OPP layers or screens.
4. Identify ambiguous terms, weak labels, or generic framing.
5. Assess whether the structure supports the Personal Command Center model.
6. Separate findings into:
   - what is working
   - what is unclear
   - what is overlapping
   - what should be refined
7. End with the clearest next product move.

# Output format

Structure the response like this:

## Product role
State what the screen or feature appears to be for.

## What is working
List the strongest parts of the structure.

## What is unclear
Call out weak hierarchy, unclear purpose, or ambiguous terminology.

## What overlaps
Identify where this screen or feature duplicates another part of OPP.

## What to refine
Give practical refinement directions.

## Recommended next move
State the best next step clearly and directly.

# Constraints

Do not:
- drift into generic UI praise
- suggest random feature additions without structural reason
- evaluate only visual polish while ignoring product meaning
- frame OPP like a normal productivity app
- recommend changes that blur the distinction between core layers
